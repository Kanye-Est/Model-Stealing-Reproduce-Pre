## Part 1 · 背景：MLaaS 与核心矛盾（3 min）
1. ML 系统三步：收集带标签数据 → 训练 f → 部署/发布（配一张 f(x)=y 的图，强调输出含 **confidence**）。
2. MLaaS 玩家：Amazon / Google / Microsoft / BigML / PredictionIO；**按查询收费**的商业模式。
3. **核心矛盾（一页讲清）**：
   - 目标 1：丰富的预测 API（高可用 + 高精度 + 置信度）；
   - 目标 2：模型机密性（货币化 + 敏感数据）；
   - 二者**根本冲突**。
4. 模型为什么要保密：商业价值 / 训练数据隐私 / 安全应用（evasion）。

> 🎤 话术：把"按查询收费"讲清楚，后面"d+1 次查询就偷走"的冲击力才出得来。

---

## Part 2 · 问题定义：什么是 Model Extraction（2 min）
- 攻击者：黑盒访问，无参数、无训练数据；目标 **f'(x) = f(x) 在 ≥99.9% 输入上成立，且查询尽量少**。
- 两个误差度量：R_test（真实分布上）、R_unif（整个空间上）。
- 三大动机：① 规避付费 ② 隐私攻击跳板 ③ evasion 跳板。
- **与传统 ML / 学习理论的区别**（关键转折点）：
  - 传统 membership query 只给标签；
  - **现实 API 多给了两样东西：置信度 + 接受部分输入** —— 这正是攻击的突破口。

---

## Part 3 · 攻击一：方程求解（6 min）★主菜
1. **二元 LR 直觉**（务必手推一遍）：
   - `f(x)=σ(w·x+β)` → 取逆 → `ln(f/(1−f)) = w·x+β` 是**线性方程**；
   - 查 **d+1** 个随机点 → 解线性方程组 → 精确还原。**非自适应、可批量**。
   - 实测：平均 41 次查询，Google 上 < $0.10。
2. **推广到 softmax / OvR / MLP**：
   - 非线性方程组 → 转成**最小化 logistic 损失**（强凸→全局最优）；
   - 作者金句 **"Noiseless Machine Learning"**；
   - ≈ 1 查询/参数；softmax 530 参数→530 查询→100%。
3. **隐私彩蛋 KLR**：representers 就是训练点 → **提取出的图像就是训练人脸/数字**（放 Fig 5）。
4. **组合拳**：extract + 白盒 model inversion，恢复 40 张脸在线查询少 **20×**（放 p13 表格）。

> 🎤 这一段是全场技术高潮，二元 LR 的取逆一定要在白板/动画上手推。

## Part 4 · 攻击二：决策树路径查找（4 min）
1. 为什么方程求解失效：树是离散分区，不是连续函数。
2. **核心 trick**：把**置信度当作叶子的"身份证"（伪标识符）**。
3. 算法直觉（配 Color/Size 玩具树图）：
   - 改一个特征值 → 看叶子 id 变没变 → 变了说明这条路径在该特征上"分裂"；
   - 连续特征二分找阈值（LINE_SEARCH），类别特征逐值试（CATEGORY_SPLIT）。
4. **top-down + 不完整查询**：从空查询逐层往下，效率更高。
5. 结果：German Credit 1,150 次查询 100% 还原；首个实用"精确"决策树提取。

---

## Part 5 · 在线实战（2 min）
- **BigML**：决策树 German Credit，1,150 查询 / 631 秒，100%。
- **Amazon**：LR + 自动特征提取（one-hot + quantile binning）。需先**逆向特征提取**，再方程求解；巧用"缺失特征置 0"构造单未知数方程。Digits 650 / Adult 1,485 查询，100%，成本 $0.07–0.15。
- **extract-and-test** 反推出 Amazon=softmax、BigML=OvR 等未公开细节。

---

## Part 6 · 防御与对抗（3 min）
- **最明显的防御 = 只返回类标签** → 引出攻击三。
- **攻击三 Label-Only**：Lowd-Meek（线性）+ 三种 retraining（uniform / line-search / **adaptive 主动学习**）。
  - 代价：比方程求解多 **~100×** 查询，但仍 >99%。**防御能涨成本，挡不住。**
- 其它对策一句话带过：
  - 置信度取整（2–3 位有点用，回归树无效）；
  - 差分隐私（不为此设计，挡不住，反例：全 0 废模型）；
  - 集成方法（可能更鲁棒）。

## 附：可能的提问与应答（备答）
- **Q：为什么置信度这么致命？** A：把"分类问题"变成"解方程问题"——一个样本从给 1 bit 标签变成给一个实数等式。
- **Q：决策树为什么不能方程求解？** A：输出对输入不连续/不可导，没有参数化的连续函数可解。
- **Q：现在的云服务还返回高精度置信度吗？** A：本文之后很多服务收敛精度 / 限流 / 加检测；可引到后续防御工作。
- **Q：和对抗样本什么关系？** A：提取是 evasion 的跳板（Papernot 同期工作：substitute model + 迁移）。
- **Q：复现和论文有差距吗？** A：核心结论一致；细节差异来自数据集划分、随机种子、求解器版本，见复现报告。
