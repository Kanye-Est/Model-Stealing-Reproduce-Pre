# 论文精读笔记：Stealing Machine Learning Models via Prediction APIs

> **出处**：Florian Tramèr, Fan Zhang, Ari Juels, Michael K. Reiter, Thomas Ristenpart. USENIX Security Symposium, 2016.
> **机构**：EPFL / Cornell University / Cornell Tech / UNC Chapel Hill
> **代码**：https://github.com/ftramer/Steal-ML
> **地位**：**模型窃取（Model Extraction）攻击方向的开山之作**，定义了问题、给出了第一批高效攻击。

---

## 0. 一句话总结

> **丰富的预测接口（Rich Prediction APIs）与模型/数据的机密性根本冲突。**
> 攻击者仅凭黑盒查询，就能用几百到几千次查询、几秒到几分钟，"偷"出一个与目标模型在 ≥99.9% 输入上一致的副本——对逻辑回归、神经网络、决策树、SVM 都成立，并在 Amazon、BigML 上实测成功。

## 1. 背景与动机

### 1.1 什么是 MLaaS（机器学习即服务）
Amazon、Google、Microsoft Azure、BigML、PredictionIO 等云服务允许：
- 用户上传数据 → 服务训练模型 → 模型以 Web API 对外提供预测；
- **可货币化（monetize）**：模型拥有者可对每次预测查询收费（Google ≥ $0.50/1000 次；BigML ≥ $0.10/1000 次；Amazon $0.0001/次）。

### 1.2 核心矛盾
| 目标 1：丰富的预测 API | 目标 2：模型机密性 |
|---|---|
| 高可用、高精度结果（含置信度） | 模型/数据货币化、训练数据敏感 |

预测接口**对外公开**，但模型本身和训练数据**机密**。模型为何值得保护：
1. **训练数据敏感**（医疗、人脸等）——模型会泄露训练数据（model inversion）；
2. **商业价值**——训练成本高，靠按查询收费摊销；
3. **安全应用**（垃圾邮件/欺诈/入侵检测）——模型泄露 = 攻击者可逃避检测（evasion）。

### 1.3 关键洞察（本文成功的根基）
现实 API 比传统学习理论假设的"只返回类标签"**泄露多得多**：
1. **返回高精度置信度值**（class probabilities），不仅是标签；
2. **接受部分特征向量**（incomplete / partial query）。

这两个"便利特性"被作者直接武器化。

### 1.4 各服务特性对照（论文 Table 1）
| 服务 | 白盒下载 | 可货币化 | 置信度 | LR | SVM | NN | 决策树 |
|---|---|---|---|---|---|---|---|
| Amazon | ✗ | ✗ | ✓ | ✓ | | | |
| Microsoft | ✗ | ✗ | ✓ | ✓ | ✓ | ✓ | ✓ |
| BigML | ✓ | ✓ | ✓ | ✓ | | | ✓ |
| PredictionIO | ✓ | ✗ | ✗ | ✓ | ✓ | | ✓ |
| Google | ✗ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

## 2. 威胁模型（Threat Model）

- **攻击者能力**：黑盒访问目标模型 f，无模型参数、无训练数据先验；可能知道也可能不知道模型类型/超参数/特征提取方式。
- **攻击目标**：用**尽可能少的查询** q，得到 f' ≈ f，在感兴趣的输入空间上**接近 100% 一致**。
- **两个误差度量**：
  - **R_test（测试误差）**：在测试集上 f 与 f' 预测不一致的比例 → 衡量"在真实数据分布上"的拟合；
  - **R_unif（均匀误差）**：在特征空间均匀采样 N=10,000 个点上的不一致比例 → 衡量"在整个空间"的拟合。
  - 比较概率分布时用 total variation 距离：R_tv = ½·Σ|y_i − y'_i|。
- **提取精度（accuracy）** = 1 − R。
- **概念区分**：
  - direct query（直接查特征向量 x）vs indirect query（查输入空间，经过特征提取 ext）；
  - **proper extraction**（提取同类模型，本文重点）vs improper extraction（用别的模型类逼近，附录证明 proper 更省）。

### 攻击动机（三类）
1. **规避查询付费 / 破坏货币化**：窃取后免费使用；或用服务的算力在大数据集上训练，再仅花 d+1 次预测就提取走（提取成本与训练集大小无关）。
2. **侵犯训练数据隐私**：作为 model inversion 的跳板（白盒 inversion 比黑盒高效得多）。
3. **模型逃避（evasion）的跳板**：拿到模型后在本地构造绕过检测的样本。

## 3. 攻击一：方程求解 Equation-Solving ★（最优雅，pre 技术重点）

**适用**：所有"带 logistic 输出层"的模型——二元 LR、多元 LR（softmax/OvR）、多层感知机 MLP。
**前提**：API 返回**置信度**。

### 3.1 二元逻辑回归（直觉引入）
模型：`f₁(x) = σ(w·x + β)`，其中 `σ(t)=1/(1+e^{-t})`。

对 sigmoid 取逆：
```
ln( f₁(x) / (1 − f₁(x)) ) = w·x + β        ← 关于未知数 (w, β) 的线性方程！
```
- 查询 **d+1 个线性无关的随机点** → 解 d+1 元**线性方程组** → **精确**恢复 (w, β)。
- **非自适应**：查询点与返回无关，可一次性批量提交。
- **结果**：所有二元数据集 R_test = R_unif = 0；平均仅 **41 次查询**，最多 113 次；概率差 < 10⁻⁹。

### 3.2 多元 LR（softmax / OvR）与 MLP
- 方程组变成**非线性**，无解析解。
- 解法：**最小化 logistic 损失**（把对方返回的概率当作软标签来"训练"同结构模型）。加一个小正则项 → 损失**强凸** → 收敛到**全局最优**（= 与 f 在所有样本上预测相同概率的 f'）。
- 作者称之为 **"Noiseless Machine Learning"（无噪声机器学习）**：和训练一个模型几乎一样，只是标签是"完美的"概率。
- 求解器：MLR 用 BFGS；MLP 用 SGD。
- **效率**：≈ **1 次查询 / 参数**。

论文 Table 4（Adult, target=Race, 5 类）：
| 模型 | 未知数 | 查询数 | 1−R_test | 1−R_unif |
|---|---|---|---|---|
| Softmax | 530 | 530 | 100% | 100% |
| OvR | 530 | 530 | 100% | 100% |
| MLP (20 hidden) | 2,225 | 11,125 | 99.96% | 99.99% |

> MLP 损失非凸、可能局部最优，需要**过定系统**（更多查询）才能逼近。

### 3.3 Kernel LR (KLR)：训练数据泄露的惊人案例
- KLR 把线性项 `w·x+β` 换成 `Σ αᵣ·K(x, xᵣ) + β`，其中 **representers xᵣ 是训练点的子集**。
- 提取模型 = 把 representers 解出来 = **直接拿到训练数据点**。
- 实验（手写数字）：提取出的 representers 在视觉上**就是训练图像本身**（论文 Fig 5a）；即使大幅低估 representer 数量，也会泄露**每个类的"平均图像"**（Fig 5b）。
- → 这是讲"模型窃取 ⇒ 隐私危害"的最佳演示。

### 3.4 应用：Extract + Model Inversion 组合拳（论文 §inversion / slides p13）
- 目标：从人脸识别模型恢复训练者的脸（Fredrikson 2015）。
- 直接黑盒 inversion：恢复 1 张脸需 20,600 次查询 / 24 分钟；恢复全部 40 人需 **800,000 次 / 16 小时**。
- 本文 **extract-and-invert**：先用 41,216 次查询提取白盒 f'，再在本地做白盒 inversion → 恢复全部 40 张脸只需 **41,000 次在线查询 / 10 小时**，在线查询量减少 **20×**，效果视觉上无差别。

## 4. 攻击二：决策树路径查找 Path-Finding ★（另一大创新）

**为何方程求解失效**：决策树不是连续函数，它把输入空间切成离散区域，每个叶子一个标签+置信度。

### 4.1 核心 idea
把每个叶子的**置信度值当作路径的"伪标识符"（pseudo-identifier / quasi-identifier）**。
- BigML 的置信度由叶子内训练样本分布算出，不同叶子往往不同 → 可作为该叶子的"身份证"。
- 通过逐个改变输入特征、观察叶子 id 是否变化，反推每条路径的判定条件。

### 4.2 算法（论文 Algorithm 1）
从随机输入 x 出发：
1. 查询得到叶子 `id = O(x)`；
2. 对每个特征：
   - **连续特征** → `LINE_SEARCH`：在该特征取值范围上二分（精度 ε），找出所有导致 id 改变的区间边界（= 分裂阈值）；
   - **类别特征** → `CATEGORY_SPLIT`：逐个取值查询，归类哪些值留在当前叶子、哪些通向新叶子；
3. 由此得到当前叶子的谓词（predicate），并生成探索其他叶子的新查询；
4. 循环直到所有叶子访问完毕。

### 4.3 top-down 变体（利用不完整查询，效率更高）
- 从**空查询**（所有特征 = ⊥）开始，查 node-identity oracle 得到根节点；
- 逐个设置特征，恰好有一个会让计算走到不同节点 → 那就是当前节点的分裂特征；
- 逐层向下递归。靠 BigML 响应里的 `fields` 字段判断节点在哪个特征上分裂。

### 4.4 工程细节
- **duplicate id**（多个叶子同 id）：检查当前查询是否违反已访问叶子的谓词来区分；
- 用到 BigML 响应的三个字段：**Prediction**（标签/值）、**Confidence**（置信度，构成 id）、**Fields**（路径上出现的特征，供 top-down 用）。

### 4.5 结果（论文 Table 6 节选）
| 模型 | 叶子数 | 唯一 id 数 | 深度 | 完整查询 1−R_test / 查询数 | 不完整查询 1−R_test / 查询数 |
|---|---|---|---|---|---|
| IRS Tax Patterns | 318 | 318 | 8 | 100% / 101,057 | 100% / 29,609 |
| German Credit | 26 | 25 | 11 | 100% / 1,722 | 100% / 1,150 |
| Email Spam | 219 | 78 | 29 | 87.2% / 42,324 | 99.7% / 21,808 |

- 不完整查询（top-down）**大幅减少查询数**、对 duplicate id 更鲁棒；
- 所有叶子 id 唯一时可**完美提取**；回归树更易（输出实值，id 几乎必唯一）。
- 与 Kushilevitz-Mansour (1993) 对比：那个只能处理布尔树、复杂度 ~2^depth 不实用；本文借助置信度做唯一标识，实现了**首个实用的"精确"决策树提取**，复杂度指数级下降。

## 5. 在线实战案例（论文 §6）

### 5.1 BigML（决策树）
- 在自己账户训练 German Credit 树（26 叶子，2 个同标签+同置信度），设为黑盒收费模型；从另一账户提取。
- Algorithm 1 用 1,722 次查询 / 1,030 秒；top-down 用 **1,150 次查询 / 631 秒**（被 ~500ms/查询的网络延迟主导）。
- 100% 还原树的路径。

### 5.2 Amazon（逻辑回归 + 自动特征提取）
- Amazon 默认开启：类别特征 **one-hot-encoding**、数值特征 **quantile binning**（按训练数据分位数分桶）。
- 攻击难点：攻击者只能在**输入空间**查询，不能任意构造特征空间方程。
- 解法：
  1. **逆向特征提取 ext**：用 line-search 在每个数值特征范围上找输出突变点 = 分桶边界（同时泄露训练数据分布）；
  2. 在本地复现 one-hot + binning；
  3. 利用 Amazon **"缺失特征 → 对应特征置 0"** 的处理，构造**单未知数方程**逐个解系数。

| 模型 | OHE | Binning | 查询数 | 时间 | 价格 |
|---|---|---|---|---|---|
| Circles | - | ✓ | 278 | 28s | $0.03 |
| Digits | - | ✗ | 650 | 70s | $0.07 |
| Iris | - | ✓ | 644 | 68s | $0.07 |
| Adult | ✓ | ✓ | 1,485 | 149s | $0.15 |

全部 100% 一致。

### 5.3 extract-and-test：逆向未知模型类型/超参数
- 对文档没说清的细节，逐一尝试不同假设、用测试集验证（只需与模型交互一次，复用样本）。
- 发现：**Amazon 用 softmax，不为缺失值建预测器；BigML 用 OvR，为缺失值建预测器**（恰好相反）。

## 6. 攻击三：仅类标签 Label-Only（最明显防御下的攻击）

若 API 只返回类标签（回到学习理论的 membership query 设定）：

### 6.1 Lowd-Meek 攻击（2005，prior work）
- 针对**线性二元分类器** `sign(w·x+β)`：用 line search 找决策边界附近的点，恢复 w, β（只能恢复**相对比例**）。
- 可扩展到**多项式核 SVM**（在变换后的特征空间做线性提取）；
- **RBF 核不行**（变换后是无限维）。

### 6.2 三种 retraining 策略（本文提出）
1. **uniform retraining**（基线）：均匀随机采 m 个点查标签，本地训练 f';
2. **line-search retraining**：用 line search 找决策边界附近的点再训练；
3. **adaptive retraining**（主动学习 / active learning）：分 r 轮，每轮在当前 f' 的决策边界附近选最"不确定"的点查询、加入训练集、重训 f'。**效果最好。**

### 6.3 结果
- 线性模型：adaptive 用 **827 次查询达 >99%**（Lowd-Meek 需 ~2,050 次，但预算足够大时更精确）。
- MLR：需 `100·c·(d+1)` 次查询达 >99.9%（平均 26,000，**比方程求解多 100×**）。
- NN、RBF SVM：更难，但大预算（~2,050 for SVM）下也能 >99%。
- **结论**：去掉置信度能把攻击成本提高约 **100×**，但**挡不住攻击**。

## 7. 对抗措施 Countermeasures（pre 讨论环节）

| 措施 | 效果 |
|---|---|
| **API 最小化**（只给标签、不响应部分查询） | 挡住方程求解 + 特征逆向，但 label-only retraining 仍可行 |
| **置信度取整 rounding** | 4–5 位无影响（BigML 本就 5 位）；2–3 位削弱但仍优于 label-only；**回归树完全无效** |
| **差分隐私 DP** | **不能阻止提取**（DP 不为此设计——反例：权重全 0 的废模型满足 DP 但可被秒解）；对模型参数加 DP 是开放问题 |
| **集成方法**（随机森林等） | 可能更鲁棒（只能得到粗近似），但仍可能被 evasion 攻击 |

## 8. 贡献总结 & 历史意义

**三大贡献**：
1. **简单的方程求解提取**（非自适应随机查询）——影响所有返回置信度的 ML 服务；
2. **新的决策树路径查找算法**——首个实用的"精确"决策树学习；
3. **仅类标签场景的攻击**——基于学习理论的 retraining，证明最明显的防御也不够。
并实测 Amazon / BigML，附带揭示训练数据泄露（KLR）与模型反演加速。

**历史意义**：开创 model extraction 方向，直接启发后续大量工作：
- 同期 Papernot et al. (2016) 用 substitute model 做对抗样本迁移；
- 后续：DNN 窃取、知识蒸馏式窃取、功能等价提取、模型水印 / 指纹、PRADA 等提取检测、以及大模型时代的 API 蒸馏与防御。
